# Composition Patterns and Originality Rules

## Basic Structure Types

Pick one structure; don't mix too many.

### Workflow / Process

Suitable for: input -> processing -> output, content production, AI workflows, automation chains.
How to draw: input on the left, Xiaohei or a weird machine processing in the middle, output on the right, orange arrows to express the main flow direction.

### System Fragment

Suitable for: information sources, filters, databases, renderers, agent system fragments.
How to draw: draw only 3-5 core modules, with Xiaohei participating in one key action.

### Before-and-After Comparison

Suitable for: chaotic/orderly, manual/automatic, scattered/consolidated, anxious/stable.
How to draw: chaos on the left, stability on the right, an orange arrow in the middle. The character can be more exaggerated.

### Character State

Suitable for: user pain points, creator states, too many tools, information anxiety, from stuck to running.
How to draw: 2-4 small states, each with one short annotation.

### Conceptual Metaphor

Suitable for: content factory, information warehouse, in-brain black box, workflow machine, automated daily report.
How to draw: one big weird object or machine, minimal input, one output. Must be memorable.

### Method Layering

Suitable for: methodology frameworks, system hierarchies, capability stacks, content system layering.
How to draw: boxes layered one by one—not a formal pyramid; Xiaohei hauling bricks or building alongside.

### Map Route

Suitable for: from idea to launch, user paths, content continuation paths, learning routes.
How to draw: one winding path, a few nodes, Xiaohei pulling a string or walking.

### Mini Comic Panels

Suitable for: failure to success, the real process, ranting, before-and-after use changes.
How to draw: 2-4 small scenes, each panel expressing only one action.

## Original Metaphor Generation Method

Reinvent the metaphor from the current article each time; don't copy old images.

### Three Steps

1. Swap the abstract concept for a physical action: getting stuck, leaking/missing, getting heavier, sorting, settling, fermenting, opening a door, folding, unpacking, backflow.
2. Swap the system structure for a low-tech object: a broken machine, cardboard box, drawer, water pipe, mailbox, weird dial, scale, well, ladder, strange workstation.
3. Let Xiaohei take on the action: not standing to the side, but stuck inside the machine, pulling the wrong string, guarding a gate, hauling, patching, weighing, holding a ladder, recording, stuffing something into some weird device.

### Usable Object Pool

- Cardboard boxes, drawers, old machines, funnels, scales, mailboxes, doors, wells, ladders, water pipes, balls of string, gates/sluices, turntables, black boxes, hole punchers, pasta rollers, clotheslines, weird workstations.
- When using, pick only 1-2; don't pile them up.

### Xiaohei Action Pool

- Pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, catch, dismantle, mark, recycle.
- Actions should serve the core meaning; don't be weird just for the sake of weird.

## Anti-Cliché Rules

Don't open or replicate `assets/examples/` by default. These images are only for style calibration—for example, line density, white space, color restraint, Xiaohei's temperament.

Unless the user explicitly says "follow this one / replicate this composition / adapt using this example," do not directly reuse these old compositions:

- Conveyor belt with two breakpoints
- Xiaohei pulling a judgment lever inside a content machine
- Xiaohei as a funnel sorting traffic/trust/conversion
- Xiaohei slicing a material fish
- Xiaohei pulling a continuation path
- Xiaohei pulling three layers of information sources
- Three Xiaohei each holding a megaphone/building a bridge/opening a door
- Xiaohei stamping a messaging toolbox
- Xiaohei holding a sign looking at a common-pitfall path

For similar themes, also switch to a new metaphor. For example, a "continuation path" doesn't have to be drawn as a route—you could draw Xiaohei attaching the content's tail to a doorknob; "one-input-many-outputs" doesn't have to be drawn as a fish—you could draw Xiaohei pressing a paper ball into several different shapes.
